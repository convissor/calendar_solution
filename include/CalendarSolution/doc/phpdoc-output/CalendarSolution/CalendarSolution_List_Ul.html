Class CalendarSolution_List_Ul

Description
Description | Vars (details) | Methods (details) Constants

The means to output collections of events formatted as an "unordered list" of the date and name of each event

Intended to show a limited number of occurrences of Featured Events on Home Pages or other Featured Pages

The limit should be established using set_limit().

get_rendering (line 138)

Produces a list of events laid out in a short list format

Cascading Style Sheet notes: the list is contained within "ul.cs_list_ul". Each event is wrapped by a "li" element, which has multiple class attributes:

  • Row: cs_row_0, cs_row_1
  • Status (Open, Full, Cancelled): cs_status_O, cs_status_F, cs_status_C
  • Changed: cs_changed_Y, cs_changed_N
  • Organizer: cs_is_own_event_Y, cs_is_own_event_N

  • return: the HTML for displaying the events
  • see: CalendarSolution_List::set_limit()
  • see: CalendarSolution_List::set_date_format()
  • see: CalendarSolution_List::set_show_own_events_frist()
  • see: CalendarSolution_List::get_limit_navigation()
  • access: public
  • uses: CalendarSolution_List::set_show_cancelled() - to drop cancelled events from the display by default. If cancelled events should be shown, call set_show_cancelled(true) before calling this method.
  • uses: CalendarSolution_List::set_permit_future_months() - to limit how far ahead people can see, but only if it has not been called yet
  • uses: CalendarSolution_List::set_where_sql() - to generate the WHERE clause and cache keys
  • uses: CalendarSolution_List::run_query() - to obtain non-cached data
  • uses: CalendarSolution::$cache - to cache the output, if possible
  • uses: CalendarSolution_List::set_from() - to default the date to today, but only if it has not been called yet
  • uses: CalendarSolution_List::set_page_id() - if "$page_id" is passed
